# Temperature Comparison using OpenRouter API

This project demonstrates how the **temperature** parameter affects responses generated by a Large Language Model (LLM). The same prompt is sent multiple times with different temperature values, allowing you to compare the variation in outputs.

## Objective

The script:

- Connects to the OpenRouter API using the OpenAI Python SDK.
- Sends the same prompt to the model with different temperature values.
- Prints the generated response for each temperature.
- Helps understand how temperature influences creativity and randomness in AI-generated text.

---

- Python 3.8 or later
- OpenRouter API Key

## Get an OpenRouter API Key

1. Create an account at https://openrouter.ai
2. Generate an API key.
3. Replace the placeholder in the script:

```python
client = OpenAI(
    api_key="YOUR_API_KEY",
    base_url="https://openrouter.ai/api/v1",
)
```

> **Note:** Never commit your API key to GitHub. Use environment variables instead.

Example:

```python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.getenv("OPENROUTER_API_KEY"),
    base_url="https://openrouter.ai/api/v1",
)
```

## Temperature Values Used

| Temperature | Behavior |
|-------------|----------|
| **0.0** | Very deterministic and consistent |
| **0.3** | Slight variation while remaining focused |
| **0.7** | Balanced creativity and accuracy |
| **1.0** | More creative and diverse responses |

## Notes

There is a small syntax error in the provided code:

Incorrect:

```python
temperature=temp,0
```

Correct:

```python
temperature=temp,
```

or

```python
temperature=temp
```

The corrected API call should be:

```python
response = client.chat.completions.create(
    model="openai/gpt-oss-20b:free",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": prompt},
    ],
    temperature=temp,
    max_tokens=300,
)
```
